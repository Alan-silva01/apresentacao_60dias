INTEGRIDADE E REPUTAÇÃO DO CANAL OFICIAL

Análise e Cuidados com a API Oficial

Controle de Duplicidade & Spam

Encontramos mais de 40 registros cadastrados com duplicidade nesta unidade. Exemplo: um mesmo nome e telefone com cadastros de *Oportunidade* e *Cliente*. Alguns contatos possuíam até 3 cadastros (*Oportunidade*, *Cliente Ativo* e *Cliente Inativo*).

  • Medida Adotada (Dashboard - Ativa): Regra manual implementada para que o sistema dispare apenas para o registro mais recente, evitando mensagens duplicadas para o mesmo telefone.
  • Medida em Análise (Próximos Passos): Pausar disparos automáticos para contatos reincidentes que não responderam a nenhuma mensagem anterior.

Validação em Tempo Real (Saldo Devedor)

Para a automação de Saldo Devedor, o sistema realiza uma consulta direta via API com o sistema EVO no milissegundo anterior ao disparo. Se o cliente tiver regularizado a pendência, o disparo é cancelado no ato, eliminando cobranças indevidas e desgaste de relacionamento.

TRANSIÇÃO ESTRATÉGICA — META 2026

Mudanças no WhatsApp

O que está mudando

A Meta já está implementando nomes de usuário no WhatsApp — em junho de 2026 já está ativo em alguns países e chega ao Brasil gradualmente ao longo do ano. Cada usuário passará a ter um ID único (BSUID) como identificador nas empresas, substituindo gradualmente o número de telefone. Quem decidir ocultar o telefone só poderá receber mensagens via BSUID.

Impacto nas automações

Nesta fase inicial, é crucial mapear o BSUID e atrelá-lo diretamente ao cadastro do cliente para garantir a continuidade dos disparos e conversas automatizadas quando a transição for 100% concluída. Cada ID é único entre cliente e empresa: se a empresa trocar o número de disparo (ou perder acesso), perde o vínculo com todos os contatos que já interagiram, exigindo cuidado redobrado.

Como estamos nos antecipando

Estamos nos antecipando analisando a estrutura de chegada desses IDs para mapear e associá-los aos cadastros dos clientes. Dessa forma, nossa arquitetura estará pronta para usar o ID ou o telefone em qualquer situação, garantindo flexibilidade total nas automações.

Como funciona o BSUID Junho 2026
Exemplo — Mapeamento BSUID
Ana Souza · (11) 99999-0000 → BR.69JN438N38U34NM3HR8
Este ID é único por empresa — o mesmo usuário terá BSUIDs diferentes para cada negócio.
Ponto de atenção: A conversa por telefone só será possível se o cliente já tiver conversado recentemente com seu contato empresarial. Se o usuário optar por usar o username (ocultando o telefone), você só conseguirá conversar com ele por esse ID. Ou seja, ele precisa ser mapeado e atrelado ao cadastro do cliente para entender o contexto e também para enviar a mensagem.

App e API no mesmo número

Antes você tinha que abandonar o aplicativo móvel para usar a API oficial. Com a Coexistência, o celular continua funcionando normalmente — você responde, faz ligações e acessa o histórico — enquanto a automação roda em paralelo via API Oficial da Meta.

  • •
    Listas de Transmissão: Disponíveis apenas através da API (são bloqueadas no aplicativo do celular ao habilitar a coexistência).
  • •
    Grupos de Conversa: As mensagens enviadas ou recebidas em grupos não sincronizam entre o aplicativo e a API.
  • •
